Principle 1

A Neuron Computes a Weighted Sum

A neuron is not a mysterious object. It is a small function.

Computation

z = w1*x1 + w2*x2 + ... + b

output = activation(z)

The lines in a neural-network diagram are learnable weights. The model learns by changing those weights.

A PyTorch nn.Linear layer is many neurons doing this calculation in parallel: y = Wx + b.

Principle 2

Layers, ReLU, and Logits

An MLP composes layers:

Two-layer example

h = ReLU(W1*x + b1)

logits = W2*h + b2

ReLU(z) = max(0, z). It sets negative values to zero.

Without nonlinear activations like ReLU, stacking linear layers is still just one linear layer. ReLU lets the model learn nonlinear decision boundaries.

Step 4

Loss, Logits, Softmax, and Cross Entropy

logits are raw, unbounded class scores.

softmax converts logits into positive probabilities that sum to one.

cross_entropy is high when the model is confidently wrong and low when confidently right.

Use raw logits for F.cross_entropy. PyTorch applies a stable log-softmax internally.

Correct

loss = F.cross_entropy(logits, y_batch)

Avoid

loss = F.cross_entropy(probabilities, y_batch)

Softmax formula: p_i = exp(logit_i) / sum_j exp(logit_j).

Principle 3

Learning Means Moving Down the Loss Surface

Think like physics: the loss is like potential energy in parameter space.

The gradient points toward the steepest increase in loss. Training moves in the opposite direction.

Step 5

The Training Loop: Epochs and Batches

seed = 42
torch.manual_seed(seed)
learning_rate = 0.01

model = ClassifierMLP(input_size, hidden_size, num_classes)
optimizer = torch.optim.SGD(model.parameters(), lr=learning_rate)

for epoch in range(num_epochs):
    model.train()
    for x_batch, y_batch in train_loader:
        optimizer.zero_grad()
        logits = model(x_batch)
        loss = F.cross_entropy(logits, y_batch)
        loss.backward()
        optimizer.step()

    validate(model, valid_loader)

Step 6

Validation Detects Overfitting

Validation does not train the model. It checks whether the model generalizes beyond the training data.

If training loss keeps falling but validation loss rises, the model is memorizing training data instead of learning a reusable pattern.

Checkpoint or stop training based on validation behavior, not just training loss.

Step 8

Prediction, AUC, and Background Rejection

For hard class prediction:

predictions = logits.argmax(dim=1)

For AUC or ROC curves, use a continuous signal score:

signal_score = logits[:, 1]

Higher signal score means the model sees the event as more signal-like.

Why accuracy is not enough

QCD background can dominate. A model can have high accuracy while missing many signal events.

AUC and rejection

AUC measures ranking quality across thresholds. Background rejection asks how much background is rejected at a chosen signal efficiency.

Project Transfer

Why JetClass Needs More Than a Plain MLP

HSF TutorialThis Project
Small tabular inputJetClass particle-level features
Signal vs backgroundTop jet vs QCD background
Simple MLPDeep Sets, PET, OmniLearn-style backbone
AccuracyAccuracy, AUC, background rejection
Basic arraysHDF5 loading, masks, fixed splits

A plain MLP expects a fixed-size, ordered input.

Jets are variable-length sets of particles. Swapping two particles should not change the jet label.

Set-based models such as Deep Sets and PET are designed for unordered particle clouds and masks.

Also normalize or scale JetClass features before training; raw physical quantities can span very different ranges.
