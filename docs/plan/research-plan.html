Research Plan · 3-Month Project

Input Feature Representations in Jet Foundation Models:
A Systematic Ablation Study

Principal Investigator: University of Toronto Physics Undergraduate (Junior) Advisor: UCL Physics Professor Target Completion: August 2026

1. Executive Summary

This project investigates how particle-level input feature representations affect the performance and transfer-learning efficiency of jet foundation models in high-energy physics. Building on the comparative study of OmniJet-α and OmniLearn, which identified input feature dimensionality as the primary determinant of peak performance regardless of architecture, this study conducts a systematic ablation of input feature configurations using the publicly available OmniLearn pretrained backbone (PET-based architecture).


The project is designed to be completed in 12 weeks by a University of Toronto physics undergraduate with strong Python and Bayesian inference skills who is building deep learning expertise. All experiments use existing open-source code and pretrained model checkpoints, requiring only single-GPU fine-tuning on cloud instances with an estimated compute budget of $100–200 USD. The output is an arXiv-ready first-author paper targeting Machine Learning: Science and Technology or a similar venue.

2. Background & Motivation

2.1 Foundation Models for Jet Physics

Foundation models—neural networks pretrained on large datasets and adapted to downstream tasks via transfer learning—are emerging as a paradigm shift in high-energy physics (HEP). In jet physics, two pioneering architectures demonstrated this capability:

  • OmniJet-α (Birk et al., 2024): GPT-style autoregressive transformer with VQ-VAE tokenization. Treats jets as ordered sequences of discrete tokens. Pretrained via next-token prediction (generation), fine-tuned for classification.
  • OmniLearn (Mikuni & Nachman, 2025): Point-Edge Transformer (PET) operating on continuous, unordered point clouds. Uses supervised pretraining on JetClass, fine-tunes across classification, generation, anomaly detection, and likelihood ratio estimation.

2.2 The Feature Representation Question

A key finding from the comparative study (Group Report, 2026) is that the dimensionality and richness of input particle representations is the primary determinant of peak performance, outweighing architectural differences between OmniJet-α and OmniLearn at dataset sizes above 103 jets. Specifically:

  • OmniLearn's performance advantage largely disappeared when its extra input features (PID, charge, energy/momentum fractions) were removed to match OmniJet-α's kinematic-only representation.
  • Discrete tokenization (VQ-VAE codebook) introduced only marginal information loss for classification.
  • In the small-data regime (≤103 jets), architectural differences persisted independently of input features—the PET architecture learned jet structure more rapidly.
  • The study called for future work "building new models which consider more particle features" to systematically understand this relationship.

2.3 Related Work on Feature Importance

Several recent interpretability studies provide complementary motivation:

  • Khot et al. (2023) used SHAP and input-zeroing ablation on constituent-based top taggers, finding that correlations between features can mislead naive importance rankings.
  • Vent et al. (2025) showed via PCA + Shapley analysis on ParticleNet that only ~5 principal components of jet substructure recover full quark-gluon discrimination AUC, suggesting significant redundancy in typical input representations.
  • Rai & Ganguly (2026) demonstrated through mechanistic interpretability of the Particle Transformer that a sparse six-head circuit recovers ~97% of AUC, with the residual stream preferentially encoding energy correlator basis functions.

These findings converge on a central open question: which input features are necessary and sufficient for foundation model performance, and how does this depend on task type and data scale?

3. Research Questions

Primary RQ: How do particle-level input feature representations affect the classification performance and transfer-learning efficiency of a PET-based jet foundation model?


RQQuestionType
RQ1 What is the minimal sufficient feature set for competitive binary top-tagging (t→bqq′ vs. QCD) using a pretrained OmniLearn backbone, and which features drive the largest marginal performance gains? Primary
RQ2 How does input feature richness modulate data efficiency—does a richer feature set reduce the amount of labeled fine-tuning data needed to reach a given performance threshold? Primary
RQ3 Does the impact of input feature configuration generalize across jet types (multi-class classification on 5 JetClass categories), or are certain features more important for specific jet classes? Secondary
RQ4 Does the feature configuration that optimizes classification also optimize generation quality (measured by Wasserstein distance), or is there a task-dependent trade-off? Exploratory

4. Methodology

4.1 Base Architecture & Pretrained Model

OmniLearn (Mikuni & Nachman, 2025), implemented in TensorFlow/Keras, using the publicly available pretrained checkpoint from github.com/ViniciusMikuni/OmniLearn. The PET backbone has on the order of hundreds of thousands of parameters—compact enough for single-GPU fine-tuning. All experiments fine-tune the pretrained backbone rather than training from scratch.

4.6 Statistical Approach

  • Uncertainty: Standard error of the mean from 3 independent repeats with different random seeds.
  • Comparison: For each data scale, compare AUC across feature configurations via ΔAUC relative to Config A (kinematic-only baseline). Report both raw values and normalized differences.
  • Data efficiency curves: For each config, plot AUC vs. log10(N). Fit a saturating exponential to estimate asymptotic performance and half-saturation dataset size.
  • Significance: Non-overlapping 2×SEM error bars as threshold for meaningful difference; supplement with Cohen's d for effect sizes between adjacent feature configs.

7. Expected Outcomes & Contributions

7.1 Anticipated Findings

  1. Feature marginal value ranking: Quantify the AUC gain per added feature category (PID, charge, energy fractions) across data scales. Expect diminishing returns beyond Config C for classification, based on the Group Report finding that additional features primarily affect peak performance at large N.
  2. Data efficiency curves: Richer features should shift the AUC-vs-N curve upward at all N, but the relative benefit should be largest at intermediate N (103–104), where the backbone has enough data to leverage feature richness but hasn't yet saturated.
  3. Task-dependent feature importance: Classification and generation may weight features differently. PID may matter more for generation (particle identity affects shower development) than for classification (kinematic substructure dominates tagging).
  4. Frozen vs. fine-tuned interaction: With richer inputs, frozen backbone performance may degrade less relative to fine-tuned, since the pretrained representation already encodes feature-rich jet structure.

7.2 Contributions

  • Empirical: First systematic ablation of input features for a jet foundation model, directly addressing a gap identified in the OmniJet-α vs. OmniLearn comparative study.
  • Methodological: A reproducible experimental framework for feature ablation in foundation model fine-tuning, applicable to future architectures.
  • Practical: Guidance for experimentalists on which particle-level features are worth the engineering cost of reconstruction in detector design and data preparation pipelines.
