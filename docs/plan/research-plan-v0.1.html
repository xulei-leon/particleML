Research Plan · 3-Month Project

Input Feature Representations in Jet Foundation Models:
A Systematic Ablation Study

Principal Investigator: University of Toronto Physics Undergraduate (Junior) Advisor: UCL Physics Professor Target Completion: August 2026

1. Executive Summary

This project investigates how particle-level input feature representations affect the binary top-tagging performance and data efficiency of jet foundation models in high-energy physics. Building on the comparative study of OmniJet-α and OmniLearn, which observed that input feature dimensionality correlated strongly with peak performance, this study conducts a systematic ablation of input feature configurations using the publicly available OmniLearn pretrained backbone (PET-based architecture), benchmarked against non-foundation baselines (Deep Sets, ParticleNet).


The project is designed to be completed in 12 weeks by a University of Toronto physics undergraduate with strong Python and Bayesian inference skills who is building deep learning expertise. The core ablation (Tier 1) requires single-GPU fine-tuning on cloud instances with an estimated compute budget of $110–130 USD; a Tier 2 extension adds ~$20 for broader scope. The output is an arXiv-ready first-author paper targeting Machine Learning: Science and Technology or a similar venue.

2. Background & Motivation

2.1 Foundation Models for Jet Physics

Foundation models—neural networks pretrained on large datasets and adapted to downstream tasks via transfer learning—are emerging as a paradigm shift in high-energy physics (HEP). In jet physics, two pioneering architectures showed this capability:

  • OmniJet-α (Birk et al., 2024): GPT-style autoregressive transformer with VQ-VAE tokenization. Treats jets as ordered sequences of discrete tokens. Pretrained via next-token prediction (generation), fine-tuned for classification.
  • OmniLearn (Mikuni & Nachman, 2025): Point-Edge Transformer (PET) operating on continuous, unordered point clouds. Uses supervised pretraining on JetClass, fine-tunes across classification, generation, anomaly detection, and likelihood ratio estimation.

2.2 The Feature Representation Question

A finding of the comparative study (Group Report, 2026) is that the dimensionality and richness of input particle representations was highly correlated with peak performance, appearing to outweigh architectural differences between OmniJet-α and OmniLearn at dataset sizes above 103 jets. Specifically:

  • OmniLearn's performance advantage largely disappeared when its extra input features (PID, charge, energy/momentum fractions) were removed to match OmniJet-α's kinematic-only representation.
  • Discrete tokenization (VQ-VAE codebook) introduced only marginal information loss for classification.
  • In the small-data regime (≤103 jets), architectural differences persisted independently of input features—the PET architecture learned jet structure more rapidly.
  • The study called for future work "building new models which consider more particle features" to systematically understand this relationship.

2.3 Related Work on Feature Importance

Several recent interpretability studies provide complementary motivation:

  • Khot et al. (2023) used SHAP and input-zeroing ablation on constituent-based top taggers, finding that correlations between features can mislead naive importance rankings.
  • Vent et al. (2025) showed via PCA + Shapley analysis on ParticleNet that only ~5 principal components of jet substructure recover full quark-gluon discrimination AUC, suggesting significant redundancy in typical input representations.
  • Rai & Ganguly (2026) showed through mechanistic interpretability of the Particle Transformer that a sparse six-head circuit recovers ~97% of AUC, with the residual stream preferentially encoding energy correlator basis functions.

These findings converge on a central open question: which input features are necessary and sufficient for foundation model performance, and how does this depend on task type and data scale?

3. Research Questions

Primary RQ: How do particle-level input feature representations affect the binary top-tagging performance and data efficiency of a fine-tuned PET-based jet foundation model, and which features drive the largest marginal gains?

Causal attribution limitation: This study reports ΔAUC and performance trends across feature configurations; it does not claim that a specific feature causes a specific performance change. Observed differences may reflect feature correlations rather than independent causal effects. Feature ablation reveals associations, not causal mechanisms.


RQQuestionType
RQ1 What is the minimal sufficient feature set for competitive binary top-tagging (t→bqq′ vs. QCD) using a fine-tuned pretrained OmniLearn backbone, and which feature categories (PID, charge, energy fractions) account for the largest marginal AUC gains? Primary
RQ2 How does input feature richness modulate data efficiency—does a richer feature set reduce the amount of labeled fine-tuning data needed to reach a given AUC threshold? Does the ranking of feature importance change with dataset size? Primary
RQ3 Does a pretrained foundation-model backbone (OmniLearn-pretrained) outperform strong non-foundation baselines (Deep Sets, ParticleNet) when both are given identical input features and fine-tuning data? Secondary
RQ4 Does the impact of input feature configuration generalize across jet types (multi-class classification on 5 JetClass categories, one-vs-rest macro-averaged AUC)? Tier 2

RQ1–RQ2 form the minimum publishable core (Tier 1). RQ3 provides the comparative context that distinguishes this work from an internal ablation. RQ4 is an extension to be pursued if Tier 1 is completed within budget and schedule. The generation task (previously RQ4) is deferred to future work (§4.3.2).

4. Methodology

4.1 Base Architecture & Pretrained Model

OmniLearn (Mikuni & Nachman, 2025), implemented in TensorFlow/Keras, using the publicly available pretrained checkpoint from github.com/ViniciusMikuni/OmniLearn. The PET backbone has on the order of hundreds of thousands of parameters—compact enough for single-GPU fine-tuning. All experiments use the fine-tuned pretrained backbone as the primary setting; from-scratch training on the same architecture serves as an auxiliary ablation (see §4.5).

4.1.1 Non-Foundation Baselines

To assess whether a pretrained foundation model backbone offers advantages over strong task-specific architectures, we include two non-foundation baselines. Both receive identical input feature configurations (Config A–D) and are trained on the same data splits, ensuring that any performance difference is attributable to architecture and pretraining, not input representation.

BaselineArchitectureReferenceImplementation
Deep Sets Permutation-invariant: per-particle MLP → sum aggregation → jet-level MLP Zaheer et al. (2017); Qu & Gouskos (2020) Re-implement in TensorFlow/Keras following the OmniLearn codebase conventions
ParticleNet Dynamic graph CNN with edge convolutions on particle point clouds Qu & Gouskos (2020), Phys. Rev. D 101, 056019 Use the Weaver reference implementation, adapted to accept the same 4-momentum input format

Note: Particle Transformer (Qu et al., 2022) is a stronger baseline in principle, but its heavier compute requirements and attention-based architecture make it less suitable as a "task-specific" control given our single-GPU budget. Deep Sets and ParticleNet span the complexity range from minimal permutation-invariant to graph-based, providing a meaningful comparison spectrum.

4.3.1 Physical Observables (Auxiliary)

Beyond classifier-level metrics, we compare distributions of physically interpretable observables to assess whether feature configuration affects the model's ability to capture jet substructure:

  • Jet mass mjet: reconstructed from the sum of constituent 4-momenta; sensitive to heavy-resonance identification (top, W, Z, Higgs).
  • N-subjettiness τ21 = τ2/τ1: discriminates 2-prong (W/Z/H) from 1-prong (QCD) substructure.
  • Jet mass distribution for correctly vs. incorrectly classified jets: checks whether models with richer features make different types of errors (e.g., failing only on jets near mass boundaries).

These are not primary metrics but provide physical interpretability: a ΔAUC of 0.01 may be statistically significant yet physically meaningless; subleading differences in jet-mass agreement may reveal which features encode physically relevant information.

4.3.2 Note on Generation Task

The earlier exploratory RQ on generation quality (Wasserstein distance evaluation) is deferred to future work. A meaningful generation comparison requires a fully specified generation head, joint-distribution metrics (e.g., MMD, FID-style score on particle clouds), and physics-level validation of generated jets—each of which significantly expands the scope. The present study focuses on the classification ablation, for which the experimental design is most mature.

4.6 Statistical Approach

4.6.1 Fixed Test Set

A single, pre-split test set is used for all experiment runs. This ensures that every model—regardless of feature configuration, backbone setting, or baseline architecture—is evaluated on exactly the same jets. Differences in metrics therefore reflect genuine model differences rather than variation in test-set composition.

4.6.2 Uncertainty Quantification

  • Bootstrap confidence intervals: For each trained model, we draw 1,000 bootstrap samples (with replacement) from the fixed test set and compute the metric on each. The 2.5th and 97.5th percentiles form the 95% confidence interval. This avoids the small-N fragility of 3-repeat SEM and makes no normality assumption about the sampling distribution.
  • Seed repeats: Each experimental condition is repeated with 5 independent random seeds for all Tier 1 (MVP) runs (see §4.5). For Tier 2 (extension) runs, 3 seeds are used. The final reported metric for a condition is the median across seeds, with bootstrap CIs pooled across the seed ensemble.

4.6.3 Formal Comparison Tests

ComparisonTestRationale
Feature Config A vs. B, C, D (same backbone, same N) DeLong test (correlated ROC curves) Same test set → ROC curves are correlated; DeLong accounts for this dependence. Used for AUC comparisons.
Accuracy comparisons (same backbone, same N) McNemar test (paired nominal data) Same test set, binary predictions → McNemar tests whether the two models disagree differently on the same instances.
OmniLearn vs. baseline model (same feature config) DeLong test Architecture comparison on identical test set with identical input features.
Data-efficiency curves (AUC vs. log N) Bootstrap CI overlap At each N, compare 95% CIs. Non-overlap indicates a meaningful difference at that data scale.

4.6.4 Data Efficiency Analysis

  • For each feature configuration, plot AUC vs. log10(N) with bootstrap 95% CIs.
  • Fit a saturating exponential f(N) = a − b · exp(−c · N) to estimate asymptotic AUC (a) and half-saturation dataset size (Nhalf = ln 2 / c).
  • Compare Nhalf across feature configurations: a smaller Nhalf for richer features would indicate higher data efficiency.

4.6.5 Effect Size Reporting

  • For each pair of adjacent feature configurations (A→B, B→C, C→D), report:
    • ΔAUC with 95% bootstrap CI
    • Cohen's d computed on the bootstrap distribution of AUC differences
    • Interpretation thresholds: |d| < 0.2 = negligible; 0.2–0.5 = small; 0.5–0.8 = medium; > 0.8 = large
  • Statistical significance is assessed at α = 0.05 with the DeLong test. We do not use non-overlapping error bars as a significance criterion.

7. Expected Outcomes & Contributions

7.1 Hypothesized Findings

The following are predictions based on prior work; the study design is agnostic to their confirmation or refutation.

  1. Feature marginal value ranking: We expect to quantify the AUC gain per added feature category (PID, charge, energy fractions) across data scales. Based on the Group Report observations, we hypothesize diminishing returns beyond Config C for classification, but this has not been systematically tested and a different pattern (e.g., PID dominating at all scales) would be equally informative.
  2. Data efficiency curves: If richer features carry complementary information to what the pretrained backbone already encodes, the AUC-vs-N curve should shift upward at all N, with the largest relative benefit at intermediate N (103–104). A null result—feature configuration having no effect on data efficiency—would suggest the pretrained representation is sufficiently expressive to compensate for missing features.
  3. Foundation model vs. task-specific baselines: We hypothesize that OmniLearn's pretrained backbone provides a performance advantage over Deep Sets and ParticleNet when both receive identical input features. However, this advantage may narrow or disappear for the richest feature configuration (Config D), if task-specific architectures are more efficient at exploiting high-dimensional per-particle information.
  4. Physical interpretability: If feature configuration affects which jets are correctly classified (as measured by jet-mass distributions of true/false positives), we expect richer features to improve classification near the W-mass peak, where PID and charge help separate hadronic W decays from QCD background.

7.2 Contributions

  • Empirical: To our knowledge, the first systematic ablation of input features for a jet foundation model, directly addressing a gap identified in the OmniJet-α vs. OmniLearn comparative study.
  • Methodological: A reproducible experimental framework for feature ablation in foundation model fine-tuning, applicable to future architectures.
  • Practical: Guidance for experimentalists on which particle-level features are worth the engineering cost of reconstruction in detector design and data preparation pipelines.
