# Review Report: CMS 2015 MINIAODSIM Feasibility Analysis

**Document reviewed:** `D:/code/particleML/docs/research/assessments/cms-2015-miniaodsim-feasibility.md`
**Review type:** Document review
**Reviewer:** OpenCode (kimi-k2.7-code)
**Date:** 2026-07-14

## Executive Summary

The document is a thorough feasibility analysis that correctly identifies CMS 2015 MINIAODSIM as a technically viable public source for the A-D feature-availability study. It appropriately scopes the work, lists required EDM products, defines the A-D feature mapping, and acknowledges the key limitation that no CMSSW event loop was run. However, several correctness and risk issues need attention before the research plan is updated and E0 is attempted. The most important gaps concern background sample composition and cross-section weighting, signal yield after truth matching, impact-parameter reference points and units, and the application of jet energy corrections.

## Findings

| Critical | Correctness / Risk | §2.2 Background, §5.4 Distribution control | No cross-section weighting is specified for combining QCD generator-level HT/pThat bins. Training on unweighted events from bins with different cross sections produces a background sample whose jet-pT composition does not reflect physical QCD production. | §2.2 lists five QCD HT/pThat bins (470-600, 600-800, 800-1000, 1000-1400, 1400-1800 GeV) with comparable event counts but very different cross sections. §5.4 only mentions pT/eta reweighting between signal and background, not inter-bin background composition. | Add an explicit rule: compute per-bin weights as generator cross section divided by number of generated events, apply them to define the background training mixture, document the weights in the extractor output, and store them in provenance. Do not feed bin labels or weights into the model. |
| High | Correctness / Risk | §8.1 Preliminary AK8 yield, §9 Yield gate | The yield estimate is only for kinematically selected AK8 jets before truth matching. | High | Clarity / Correctness | §4 A-D Feature Mapping, §9 D-fields gate | Impact-parameter reference points and units are ambiguous. `dxy()` and `dz()` in `pat::PackedCandidate` use different reference points (typically beamspot/reference point for dxy, primary vertex for dz), and the units are not stated. | §4 lists `dxy_raw` and `dz_raw` with rule "Evaluate with respect to the selected primary vertex" only for dz. §9 D-fields gate requires "documented units and plausible distributions" but the current text does not specify cm or the PV choice. | State explicitly that values are stored in cm, that `dxy` is signed with respect to the beamspot/reference point and `dz` with respect to the leading offline primary vertex, and add validation histograms comparing PV z position and beamspot position distributions. |

## Overall Verdict

**Conditional approval.** The document is a sound starting point for the E0 phase, but the four High/Critical findings (QCD cross-section weighting, signal yield estimate, impact-parameter reference points, and jet energy corrections) must be addressed in the next revision of the research plan and in the E0 pilot design before bulk extraction begins.
